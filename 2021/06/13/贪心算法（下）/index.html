贪心算法（下）

笔试和面试的目的是不一样的，笔试要的是通过率，面试要的是区分度。再难的公司笔试五道题能够100%AC两道题一般就有面试资格，可能会因为简历卡你，但是不会因为笔试成绩卡你。

而面试要的区分度是，一道题你给出了暴力的算法，可能低分，优化算法中分，很好的解高分。而贪心策略题目的成败完全在于你贪心策略想没想对，所以它有很好的淘汰率，但是没有很好的区分度。一道题你贪心策略相对了就是100分，没想对就是0分，这种零和题目是没有多少区分度的。所以在笔试唯结果论的情况下当然是可以用对数器进行偷巧的。如果面试靠你贪心算法那你就要证明了，当然你也可以先用对数器证明其的正确性。

贪心算法例题1：会议的最佳安排

一些项目要占用一个会议室宣讲，会议室不能同时容纳两个项目来宣讲，给你一个项目的开始和结束时间，怎样安排会议才能使宣讲场次达到最多。

这道题的贪心思路有很多，比如先安排会议时间短的，先安排时间开始早的等等。这道题的结论是先安排会议结束时间早的能够使安排数尽可能地多

//纯暴力解题过程：
//还剩下的会议都放在programs[]里
//done是之前已经安排了多少会议
//timeline是目前来到的时间点

//目前来到了timeline的时间点，已经安排了done多的会议，剩下在programs[]里的会议可以自由安排
public static int process(Program[] programs, int done, int timeline) {
    if(programs.length == 0) {
        return done;
    }
    //还有会议可以选择
    int max = done;
    //当前安排的会议是什么会议，每一个都枚举
    for(int i = 0; i < programs.length; i++) {
        if(programs[i].start >= timeline) { //start>= 是时间晚于于的意思，也就是你当前会议的开始时间晚于当前来到的timeline
            Program[] next = copyButExcept(programs, i);
            max = Math.max(max, process(next, done+1, programs[i].end));
        }
    }
    return max;
}

//copyButExcept方法是将会议从Program[]中取出的方法
public static Program[] copyButExcept(Program[] programs, int i) {
    Program[] ans = new Program[programs.length - 1];
    int index= 0;
    for(int k = 0; k < programs.length; k++) {
        if(k != i) {
            ans[index++] = programs[k];
        }
    }
    return ans;
}

贪心算法的代码：

public static int bestArrange2(Program[] programs) {
    Arrays.sort(programs. new ProgramComparator()); 
    //自己定义一个比较器，谁结束时间早谁排在前面（就是把return那里给修改一下即可，这个不懂的前面文章写过比较器与堆，自己去看）
    int timeline = 0;
    int result = 0;
    for(int i = 0; i < programs.length; i++) {
        if(timeline <= programs[i].start) {
            //如果当前时间早于会议开始时间，则可以安排，晚于的话安排不了
            result ++;
            timeline = programs[i].end;
        }
    }
    return result;
}

贪心算法例题2：墙与灯

一个街道上有墙（用X代表）与居民楼（用 · 代表）。居民楼的位置可以放置灯，每一个灯可以照亮其左右两边的居民楼，墙无法用来放置灯，也无需被照亮，问：如何使用最少的灯数来照亮所有的居民楼？

暴力解法：直接遍历每一个位置放灯和不放灯时的所有组合，并返回满足可以照亮所有居民楼的结果。

public static int minLight1(String road) {
    if(road == null || road.length() == 0) {
        return 0;
    }
    return process(road.toCharArray(), 0, new HashSet<>());
}
// str[index..]位置，自由选择放灯还是不放灯
//str[0..index-1]位置呢？已经做完决定了，那些放了灯的位置，存在lights里
//要求选出能照亮所有.的方案，并且在这些有效方案中，返回最少需要几个灯
//HashSet<>()里装的是我放灯的位置
public static int process(char[] str, int index, HashSet<Integer> lights) {
    if(index == str.length) { //index到达街道末尾，也就是放灯结束的时候
        //检查是否满足所有居民楼被照亮，以及返回此时所使用的最少的灯数
        for(int i = 0; i < str.length; i++) {
            if(str[i] != 'X') { //当前位置是点的话
                if(!lights.contains(i - 1) 
                   && 
                   !lights.contains(i) 
                   && 
                   !lights.contains(i+1)) {
                    return Integer.MAX_VALUE; //返回整数的最大值以当前结果无效代表无效
                }
            }
        }
        return lights.size();
    } else { //str还没结束，i位置还可以选择放灯或不放灯
        int no = process(str, index + 1, lights); //i位置不管是X还是·都有这种选择，都可以选择不放灯
        
        int yes = Integer.MAX_VALUE;  //只有i位置是点的时候才可以不放灯
        if(str[index] == '.') {
            lights.add(index);
            yes = process(str, index + 1, lights);
            lights.remove(index);
            //解释一下这里为什么要remove，这个比较抽象，就是你每个位置可以放灯也可以不放灯，测试完一次之后要清空，继续下一次所有位置放灯或者不放灯（测试完一种情况之后要清空现场，否则下一次测试会受到本次测试结果的影响）（为什么不是同一次？同一次放灯的测试中在process时已经进行完毕，能执行到remove代表其所有清空已经遍历完了）
        }
        return Math.min(no, yes);
    }
}

上面那个remove现场的情况，举个例子：

这里0位置可以放灯也可以不放灯，0位置不放灯的情况还可以继续往下遍历，即1位置是否放灯，那上面0位置也可以放灯，我们在指向process的过程中始终用的是一个lights，没有进行复制，那么你在0位置没放灯，1位置放了灯之后，想返回上面做0位置放了灯的情况，此时lights是有1位置放灯的，那你不删掉1位置的话，接下来在0位置放灯，1位置不放灯这种情况就无法获取到，即不清除现场的话可能会影响我们的后续处理。如果你每一次都克隆出一个新的就不用考虑清楚现场的问题了。

贪心解法：

i位置可能是X或者是 · ，当其是X的时候在i+1位置做决定

当i位置为点的时候，分两种情况：

  1. i+1位置是X，i位置必放灯，然后在i+2位置做决定
  2. i+1位置是点，必放i+1位置，剩下的就在i+3位置做决定，然后又分为i+3位置是X是 · ，重复以上两部判断
//代码如下：
public static int minLight2(String road) {
    char[] str = road.toCharArray();
    int index = 0;
    int light = 0;
    while(index < str.length) {
        if(str[index] == 'X') { //如果i位置是X的情况，直接跳到下一个位置做决定
            index++;
        } else {  //i位置不是X，是点
            light++;  //走i位置是点的两种情况都是要放灯的
            if(index + 1 == str.length) {
                break; //如果已经到达末尾，直接break
            } else {
                if(str[index + 1] == 'X') {
                    index = index + 2;  //i+1位置是X，在i+2位置做决定
                } else {
                    index = index + 3;  //i+1位置是点，在i+3位置做决定（灯上面light++都放过了）
                }
            }
        }
    }
    return light;
}

贪心：总是在当前状况下用我的策略决定当前步数的最优，情况变化之后再找下一个情况下的最优

贪心算法例题3：切割金条

一块金条切成两半，是需要花费与其长度数值一样的铜板的。

比如金条长度为20，那不管怎么切，都要花费20个铜板。现在一群人想瓜分整块金条，怎样切最省铜板？

例如：给定数组[10，20，30]，代表一共3个人，整块金条长度为60，金条要分成10，20，30三个部分

如果先把长度60的金条分成10和50，花费60；再把长度50的金条分成20和30，花费50；一共花费110铜板。

但如果先分成30和30，花费60；在将其中一个30划分成10和20，花费30；一共花费90铜板。

小根堆，大根堆，排序是贪心策略最常用的三个手段，为什么？堆天然就具备把样本根据我的标准排个序之后组成一个堆，然后依次弹出样本的特点。排序也是一样。这也是为什么笔试不喜欢考贪心，因为你提出贪心策略之后，来个堆来个排序，代码就结束了。

本题的贪心最优解：尽量平均。那么怎么做呢？

先把所有要划分成的结果放到小根堆里去，然后使用哈夫曼树

堆中弹出两个数字，相加过后放进去，再弹出两个数字，相加过后放进去，直到放入最后一个数字时，停。此时所有非叶子结点相加起来的总和就是最小花费。（别问为什么，这就是贪心最优解，不用花时间去证明，证明贪心就是一个坑，左神花了3个月，发现是个坑）贪心就是长见识积累题型敢去试。不证了，因为不重要。你要证你自己去证吧，我要干其它的了。

下面来看代码：

暴力代码，process去穷举任意两个结果中的金条长度去加在一起求后续，每一次都这样。

贪心代码，准备一个小根堆，把所有的数扔进去，每一次弹出两个数，合成一个数，它一定是非叶子结点，把它累加到sum上去，再把合成的数扔到小根堆里。周而复始，直到小根堆里只剩下一个数的时候，结束。

//贪心代码：
public static int lessMoney2(int[] arr) {
    PriorityQueue<Integer> pQ = new PriorityQueue<>(); //这个是系统提供的数据结构队列，默认是小根堆
    for(int i = 0; i < arr.length; i++) {
        pQ.add(arr[i]);
    }
    int sum = 0;
    int cur = 0;
    while(pQ.size() > 1) {
        cur = pQ.poll() + pQ.poll();
        sum += cur;
        pQ.add(cur);
    }
    return sum;
}

贪心算法例题4：项目的最大利润

现在你可以做项目，每个项目有对应的启动资金和利润，例如启动资金是1，利润是3，那么你总共赚了4。现在给定你每个项目的启动资金W=1和你能做的最多的项目个数上限K=2，求该怎么做项目才能让自己拥有的财富最多。（每个项目只能做一次）。贪心策略是最贴近自然智慧的算法，你多想一下，可能就会有想法的。

本题最优贪心解如下：先创建一个小根堆，把所有的花费放进去（从小到大）。它还有一个名字，叫此时被锁住的项目。你再准备一个大根堆，这个大根堆按利润组织，它还有一个名字叫做解锁的项目。好，我要开始了，都让开，要开始装逼了。

初始资金W=1，小根堆中所有初始资金能够cover的项目都弹出来，放入大根堆。做初始资金解锁完之后的项目，做哪个？大根堆堆顶的这个，即(1,3)。此时你的初始资金变成4，接下来拿着初始资金再去小根堆里解锁，在大根堆里做。如此往复做够k个。
