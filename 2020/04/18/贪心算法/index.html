贪心算法

先把上节没做完的二叉树的题给做了…

例题：给定一个二叉树的头结点head，返回这棵树是不是完全二叉树

方法一：用队列对二叉树进行宽度优先遍历(层次遍历)

public boolean isCBT1(Node head) {
    if (head == null) {
        return ture;
    }
    LinkedList<Node> queue = new LinkedList<>();
    //是否遇到过左右两个孩子不双全的结点
    boolean leaf = false;
    Node l = null;
    Node r = null;
    queue.add(head);
    while(!queue.isEmpty()) {
        head = queue.poll();
        l = head.left;
        r = head.right;
        if (
            //如果已经遇到了不双全的结点(叶子结点)之后，又发现当前结点不是叶子结点
            (leaf && (l != null || r != null)) 
            || 
            //左孩子为空但是右孩子不为空
            (l == null && r != null)
        ) {
            return false;
    }
        if(l != null) {
            queue.add(l);
        }
        if(r != null) {
            queue.add(r);
        }
        if(l == null || r == null) {
            leaf = true;
        }
}
    return true;
}

方法二：用二叉树的递归套路

判断以x结点为头结点的树是不是完全二叉树，怎么分类呢？这个分类标准很有意思，根据最后一层的结点到哪了来分类。

  1. 所有层都是满的，没有缺口
  2. 最后一层有缺口
    • 缺口停在左树上
    • 缺口停在右树上

我们需要向左右子树索要的信息：1. 是否是满的 2. 树的高度

  1. 什么情况下整体是满的？左树满，右树满，且左右子树高度一样。
  2. 第二种的情况1，左树是完全二叉树，右树是满二叉树，并且左树的高度比右树大一个
  3. 第二种的情况2，左树是满二叉树，右树是完全二叉树，并且左树的高度和右树相同

所以，对于任何一棵子树，都需要返回三个信息：

  1. 是否是满二叉树
  2. 是否是完全二叉树
  3. 高度
public static class Info {
    public boolean isFull;
    public boolean isCBT;
    public int height;
    
    public Info(boolean full, boolean cbt, boolean h) {
        isFull = full;
        isCBT = cbt;
        height = h;
    }
}

public static Info process(Node X) {
    if(X == null) {
        return new Info(true, true, 0);
    }
    Info leftInfo = process(X.left);
    Info rightInfo = process(X.right);
    //向左右两边要完信息我要想办法整合出自己的信息来，整个递归才能连起来
    int height = Math.max(leftInfo.height, rightInfo.height) + 1;
    
    boolean isFull = leftInfo.isFull && rightInfo.isFull && leftInfo.height == rightInfo.height;
    
    boolean isCBT = false;
    if(isFull) {
        isCBT = true; //满二叉树一定是完全二叉树	
    } else { //以x为头整棵树，不满  
        if(leftInfo.isCBT && rightInfo.isCBT) {//注意第二种情况的两种小情况左右子树都是完全二叉树
           
            if(leftInfo.isCBT && rightInfo.isFull && leftInfo.height = rightInfo.height + 1) {
                isCBT = true;
            }
            if(leftInfo.isFull && rightInfo.isCBT && leftInfo.height = rightInfo.height) {
                isCBT = true;
            }
        }
    }
    return new Info(isFull, isCBT, height);
}

下面来一道稍微难一点的题：

给定一棵二叉树的结点head，和另外两个结点a和b。返回a和b的公共祖先

这个题和上题一样也可以分两种做法：用套路和不用套路做。不用套路做就遍历整棵树，把欲求结点从下往上遍历包括其所有的父结点都加入到HashSet中，然后再用另一个结点从下往上依次遍历看其父结点是否在HashSet中，即可找到他们的公共祖先

用二叉树的递归套路（假设它们的公共祖先结点是x）：

  1. 结点o1和o2都不在以x为头的树上（直接不成立）
  2. 只有一个在x上（不成立）
  3. o1，o2都在以x为头的树上
    1. 左右各一个
    2. 左o1，o2
    3. 右o1，o2
public static class Info {
    public Node ans;
    public boolean findO1;
    public boolean findO2;
    
    public Info(Node a, boolean f1, boolean f2) {
        ans = a;
        findO1 = f1;
        findO2 = f2;
    }
}

public static Info process(Node X, Node o1, Node o2) {
    if(X == null) {
        return new Info(null, false, false);
    }
    Info leftInfo = process(X.left, o1, o2);
    Info rightInfo = process(X.right, o1, o2);
    
    boolean findO1 = X == o1 || leftInfo.findO1 || right.findO1;
    boolean findO2 = X == o2 || leftInfo.findO2 || right.findO2;
    
    //o1和o2最初的交汇点在哪
    Node ans = null;
    if(leftInfo.ans != null) {
        ans = leftInfo.ans; //如果左树已经发现交汇点了，肯定比x发现的早，以左树上的答案为准
    }
    if(rightInfo.ans != null) {
        ans = leftInfo.ans; //如果右树已经发现交汇点了，肯定比x发现的早，以右树上的答案为准
    }
    if(ans == null) {
        if(findO1 && findO2) {
            ans = X; //如果左树也找到了o1，右树也找到了o2，但是交汇点不在两子树上，x为交汇点
        }
    }
    return new Info(ans, findO1, findO2);
}

这道题的代码确实有点绕…不过二叉树的递归套路确实好用。这个套路是很强的，树型DP的所有问题都可以用这个套路解。

贪心算法

下面正式进入正题：贪心算法
  1. 最自然智慧的算法
  2. 用一种局部最功利的标准，总是做出在当前看来是最好的选择
  3. 难点在于证明局部最功利的标准可以得到全局最优解(一步一步把我的阶段性目标达到，最后能达到总的最优).如果你的贪心算法最后不会错过全局最优解，那么它就是有效的，反之就是无效的

那你怎么证明一个看不出门道的贪心算法是好的贪心算法呢？下面这道题从头到尾连证明方法也讲一遍，当然，你在面试的时候不要去证明贪心算法：

从头到尾讲一道利用贪心算法求解的题目：

给定一个由字符串组成的数组strs，必须把所有字符串拼接起来，返回所有可能的拼接结果中，字典序最小的结果

有暴力方法，但是时间复杂度太高，贪心用compareTo方法即可。这题不用了解，只听证明过程即可。证明贪心算法的比较过程并不是让你以后每次使用贪心算法的时候都去进行证明，太麻烦了。

贪心算法求解的标准过程
  1. 分析业务
  2. 根据业务逻辑找到不同的贪心策略(在每一步都找到对于当前步来说最合适的决策)
  3. 对于能举出反例的策略直接跳过，不能举出反例的策略要证明有效性

这往往是特别困难的，要求数学能力很高且不具有统一的技巧性。每一道专属的贪心算法都要去证明的话，这个其实是我们无法接受的。

大佬的贪心算法”偷巧”解题策略

还记得以前讲过的对数器吗小朋友？怎么样，是不是恍然大悟！我直呼大妙。

  1. 实现一个不依靠贪心策略的解法X，可以用最暴力的尝试
  2. 脑补出贪心策略A、贪心策略B、贪心策略C…
  3. 用解法X和对数器，用实验的方式得知哪个贪心策略正确
  4. 不要去纠结贪心策略的证明

听起来很不炫酷，但实际上它彻底地给你省掉了证明的过程。芜湖~起飞。

贪心在笔试中占比很高，六到七成，而且每一道贪心算法的证明是专属于那个题的，你只有研究清楚那个题的业务你才清楚该怎么证。可能你刷了再多题遇到一个新的贪心你还是得重新想。

贪心在面试中出现的比重低于20%

小根堆，大根堆，排序，是贪心策略最常用的三个东西
