并查集

  1. 有若干个样本a、b、c、d…假设类型都是V

  2. 在并查集中一开始认为每个样本都在单独的集合里

  3. 用户可以在任何时候调用如下的两个方法：

    boolean isSameSet(Vx, Vy)：查询样本x和样本y是否属于同一个集合

    void union(Vx, Vy)：把x和y各自所在集合的所有样本合并成一个集合

  4. isSameSet和union方法的代价越低越好（最好O(1)）

现在认为每个点往上指，往上到不能再往上的那个点就代表这个集合。（代表点一开始都是自己指向自己的）

查询是否是同一个集合的逻辑：（isSameSet）

给定的需要查询是否在同一个集合中的两个点，分别往上找，找到最上面的甲乙两点，判断甲是否等于乙，如果等于则X与Y在同一个集合中，反之则不在。

union合并集合的逻辑：

并查集一共有三个表：

  1. node表，a对应node-a（圈圈里面一个a）
  2. parent表，里面存放了上下结点的关系
  3. sizeMap，只有代表点才有，代表这个代表点下总共连着多少个点

你先把所有的样本给我，然后我该如何给你初始化呢？我先把所有的样本包一层node，每一个点最开始的parent都是自己，代表点也是自己。

注意：sizeMap里只留代表点的记录：比如下图，sizeMap中只有asizeMap的记录是4，bcd都没有记录，因为没必要

但是注意一个小点：这个表还是得区分按值传递还是按引用传递。按值传递没法搞出两个一样的值为3的结点，引用传递就可以。这里先不谈，这是具体实现细节的改动，不属于并查集算法的内容。

findFather方法：(往上找父结点的方法）最上面的结点其自己就是自己的父结点。

然后把所有的点的代表点改成最上面那个点。（相当于把这个链下面的所有子结点直接改成d的直接子结点，这样以后查找效率直接起飞），这样d刚好也是所有点的代表点

isSameSet与union合并方法：（这两个方法都比较好理解）

所以说搞这个并查集干嘛？除了findFather过程，其它的所有方法都是O(1)的，所以并查集的瓶颈集中在findFather上，所以把并查集扁平化，可以大幅提高效率。

两个重要优化：

  1. 把链扁平化，查找时只难受第一次，减少遍历链的高度
  2. 小挂大，也是减少遍历链的高度

接下来一个魔性的问题来了：复杂度怎么算？find方法调用得越多，性能优化越好，调用次数超过N之后，单次调用得平均代价会变成O(1)。有个大神得到了这个结论，但是用数学证不出来，1964年证到1989年，证了25年才搞定，所以不要纠结为什么有这个结论，调用频繁且单次调用O(1)，记住这个结论即可。算法导论第23章都在搞这个东西，英文版的写了快tm25页，人都看没了，但是不妨碍你用它。

union代码还可以更简洁一点：

例题：给你一堆东西，判断有多少个不同的人。任何一个属性相同就认为它们是同一个人。

直接往并查集里放，sizeMap里有多少就是多少个人。

//(1,10,13)  (2,10,37) (400,500,37)
//如果两个User，a字段一样、或b字段一样、或者c字段一样，就认为是一个人
//请合并Users，返回合并之后的用户数量
public static int mergeUsers(List<Users> users) {
    UnionSet<User> unionFind = new UnionSet(Users); //先把所有的user放入并查集里面去
    HashMap<String, User> mapA = new HashMap<>();
    HashMap<String, User> mapB = new HashMap<>();
    HashMap<String, User> mapC = new HashMap<>();
    //三个Map分别放着上面对应的三个字段
    
    for(User user : users) {
        if(mapA.containsKey(user.a)) {
            unionFind.union(User, mapA.get(user.a));
        } else {
            mapA.put(user.a, user);
        }
        if(mapB.containsKey(user.b)) {
            unionFind.union(User, mapB.get(user.b));
        } else {
            mapB.put(user.b, user);
        }
        if(mapC.containsKey(user.c)) {
            unionFind.union(User, mapC.get(user.c));
        } else {
            mapC.put(user.c, user);
        }
    }
    //向并查集询问，合并之后，还有多少个集合？
    return unionFind.getSetNum();
}
