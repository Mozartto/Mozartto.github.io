二叉树的先序、中序、后序遍历。这个是数据结构的基础概念，这里就不展开了。不会的可以查资料、看课本。

但是教科书上讲的这个先序中序后序有点low。先序中序后序遍历本质上就是打印行为在方法中的位置不同，所以先序中序后序根本就不是递归遍历的本质，递归遍历的本质是递归序。

下面我们来讲讲什么是递归序。这个很重要，比你想象得要重要。后面讲的树型DP问题(在树上做动态规划)，其实全部基于这个认知，只是你们老师啥都没给你们讲。

递归序的本质

递归序的本质，每一个结点都会来到3次（以二叉树的遍历为例，如上图）。那什么是先序呢1？你第一次来到一个结点时就打印，为先序(如下图)。你第二次到达时再打印，就是中序，你第三次到达时再打印，就是后序。任何一个结点都能在它的左树上收集一些信息，也能跑去右树上收集信息，还能第三次在它自己身上做整合，这是在树上做动态规划的基础。

使用非递归方法实现遍历——压栈实现

使用压栈法实现先序遍历的步骤：1.先打印 2.如有右，压入右 3.如有左，压入左

public static void pre(Node head) {
    System.out.print("pre-order：");
    if(head != null) {
        Stack<Node> stack = new Stack<Node>();
        stack.add(head);
        while(!stack.isEmpty()) {
            head = stack.pop();
            System.out.println(head.value +  " ");
            if(head.right != null) {
                stack.push(head.right);
            }
            if(head.left != null) {
                stack.push(head.left);
            }
        }
    }
    System.out.println();
}

如果是想使用“头、右、左”的顺序，把后两步压栈顺序改一下就行了。

后序遍历的第二种算法：（只使用一个栈）

条件1：整条左边界依次入栈；条件2：条件1的执行条件不满足时，弹出结点并打印

只使用一个栈的后序遍历算法：

//先准备好两个引用，h和c
//然后准备了一个栈
//也就是说我们看流程的时候只需要看三个变化，一个是h，一个是c，一个是栈
public static void pos2(Node h) {
    System.out.print("pos - order: ");
    if(h != null) {
        Stack<Node> stack = new Stack<Node>();
        stack.push(h);
        Node c = null;
        while(!stack.isEmpty()) {
            c = stack.peek();
            if(c.left != null && h != c.left && h != c.right) {
                stack.push(c.left);
            } else if(c.right != null && h != c.right) {
                stack.push(c.right);
            } else {
                System.out.print(stack.pop().value + " ");
                h = c;
            }
        }
    }
    System.out.println();
}

来说一下while中的三个if逻辑分支的作用。首先，h的作用是什么呢？注意观察，每次打印完当前结点c之后都会把c的值赋值给h，h的作用就是记录已经打印过的上一个结点。然后我们来看第一个if，c的左孩子不为空且h不是c的左右孩子，说明c的左右孩子还没有打印过，先遍历左子树（本代码为后序遍历），如果前两条逻辑分支中了，就代表c的左孩子已经被打印过了，短路与不会看后面的最后一个条件，所以下面看c的右孩子是否被打印过(h是否被记录为c的右孩子)，如果c的右孩子没有被打印过则遍历c的右子树。如果c的左右子树都被遍历过或都为空，则打印c本身。注意：如果你没有打印过任何结点，你只需要保证h处在一个不会干扰到你的位置就可以了。在还没有打印树中的任何结点之前，h是不会干扰逻辑分支1里面的条件判断的

如果此时结点4已经打印完毕，h处于结点4的位置，那么它此时处的位置4是不会干扰到你接下来要打印的5这个逻辑分支的。注意第一条逻辑分支中判断左树有没有处理过的时候也要判断h是不是c的右子结点，为什么？看下图这种情况，此时c的右子结点刚好打印完，此时h处于c的右子结点的位置。那么前两段关于左子结点的判断是可以通过的（左子结点不为空且h处于c的左子结点上），但是此时h在c的右子结点上，右子结点都处理完了左子结点肯定也处理完了，所以在判断第一条左子结点时要加入右子结点的判断。你打印了往上，就得了。

实现二叉树的按层遍历

  1. 其实就是宽度优先遍历，用队列
  2. 可以通过设置flag变量的方式，来发现某一层的结束（看题目）
  3. 整体思路很简单，先弹出当前结点打印，并依次加入弹出结点的左子结点和右子结点

现在请你统计二叉树的最大宽度（那一层的结点最多，并统计这一层的结点数量）。这要求我们不仅能够完成按层打印，还要能发现每一层的开始和结束。求树的最大宽度是一道非常经典的面试题。

public static class Node {
    public int value;
    public Node left;
    public Node right;

	public Node(int data) {
        this.value = data;
    }
    //第一种方法：用map的方式
    public static maxWidthUseMap(Node head) {
        if(head == null) {
            return 0;
        }
        Queue<Node> queue = new LinkedList<>();
        queue.add(head);
        //key在哪一层，value
        HashMap<Node, Integer> levelMap = new HashMap<>();
        levelMap.put(head, 1); //头结点在第一层我们都知道
        int curLevel = 1;  //当前结点在哪一层
        int curLevelNodes = 0; //当前层的结点数默认弹出之后再计入数据，进去的时候先不计数
        int max = 0;  //max就是你更新的所有层的宽度中的最大值
        while(!queue.isEmpty()) {
            Node cur =  queue.poll(); //当前队头结点出队并赋给cur(current:当前的)
            int curNodeLevel = levelMap.get(cur); //从levelMap中把当前结点在的层数给取出来
            if(cur.left != null) {
                levelMap.put(cur.left, curNodeLevel + 1); //如果左孩子不为空，加入左孩子并标记左孩子所处																						的层数
                queue.add(cur.left);  //左孩子入队
            }
            if(cur.right != null) {
                levelMap.put(cur.right, curNodeLevel + 1);
                queue.add(cur.right); //如果右孩子不为空，记录右孩子的层数并将右孩子入队
            }
            if(curNodeLevel == curlevel) {//如果当前节点的层数和你统计层的层数一样，说明此结点就在此层
                curLevelNodes++; //当前层结点数++,当前层没有结束，所以我就不更新max了
    //如果没有中这个逻辑分支，说明此时1的子节点已经入队，说明上一层该结算了		
                
            } else {
                max = Math.max(max, curLevelNodes); //找到了目前为止的最大宽度,也就是上一层结算出来的宽度
                curLevel++;  //当前层成为下一层(第二层)
                curLevelNodes = 1; //当前层弹出了第一个结点，当前层结点数设置为1
            }
    }
   max = Math.max(max, curLevelNodes); //最后一层再也没有新层去结算它了，所以要在while结束之后再更新一次max
}

不使用HashMap的方法：

下面来看一下不用HashMap的方法：

public static int maxWidthNoMap(Node head) {
    if(head == null) {
        return 0;
    }
    Queue<Node> queue = new LinkedList<>();
    queue.add(head);   //准备一个队列和几个变量
    Node curEnd = head; //当前层最右结点是谁
    Node nextEnd = null;//下一层最右结点是谁
    int max = 0; //我只需要知道当前层什么时候结束然后更新max
    int curLevelsNodes = 0; //这个有没有都可以，因为我只需要知道当前层什么时候结束然后更新max
    while(!queue.isEmpty()) {
        Node cur = queue.poll();
        if(cur.left != null) {
            queue.add(cur.left);
            nextEnd = cur.left;//当前层在不断更新下一层的最右结点，只要下一层出来一个结点，下一层的最右结点就会更新
        }
        if(cur.right != null) {
            queue.add(cur.right);
            nextEnd = cur.right;
        }
        curLevelNodes++;
        if(cur === curEnd) { //如果当前结点是当前层最右的结点，开始结算
            max = Math.max(max, curLevelNodes);//更新当前的最大宽度
            curLevelNodes = 0; //当前层结算完跳到下一层，下一层的结点是0个(目前在当前层的最右结点上，下一层																				的结点还没有弹出)
            curEnd = nextEnd;//下一层的最右马上就要变成当前层了，所以我设置下一层的最右结点
            
        }
        //注意上面的一行代码，当前层再不断为下一层的最右结点做准备，以便我这一层结束之后，下一层的结点可以直接变成当前层的结点，就是用这种机制，完成的每一层的发现和结束的时刻
    }
    return max; //不用在while后面再更新max，因为我当前层结束当前层直接结算，不用等到下一层结算
}

二叉树的序列化和反序列化

  1. 可以先用先序或者中序或者后序或者按层遍历，来实现二叉树的序列化
  2. 用了什么方式序列化，就用什么方式反序列化

（在树上是可以玩KMP的，这波神仙操作后面听左神讲…）

序列化的时候把空节点加上。反序列化的方式和序列化的方式一样，先头再左再右，先把1拿出来建头结点，然后把1拿出来建左节点，然后把空拿出来建左节点，返回，把1拿出来建右结点….以此类推

中序和后序就是把Node head那一句分别加在左和右中间、最后
